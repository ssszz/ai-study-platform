## 一、项目概述

### 1.1 背景

部门同事对 AI 领域尚不熟悉，需要一个内部学习与考核平台，帮助大家系统掌握 AI 基础知识以及金蝶灵基（Lingee）平台的使用技能。

### 1.2 目标

构建一个集学习、练习、考试、评分、分析于一体的 AI 学习平台，支持多等级、多领域的知识体系，让同事能够循序渐进地学习和检验 AI 技能。

### 1.3 用户角色

| 角色 | 权限 |
|------|------|
| **普通用户 (user)** | 学习课程、参加考试、查看自己的成绩与错题 |
| **管理员 (admin)** | 上述全部 + 题库管理、课程管理、用户管理、创建考试、部门成绩分析 |

### 1.4 技术栈

| 层 | 技术 |
|---|------|
| 前端 | React 18 + TypeScript + Tailwind CSS + Vite |
| 后端 | Python 3.11+ + FastAPI + SQLAlchemy |
| 数据库 | SQLite（单文件，便于迁移） |
| 认证 | JWT Token + bcrypt |

## 二、功能模块

### 2.1 首页仪表盘

- 欢迎语 + 个人统计卡片：已学课程数 / 总数、考试次数、平均得分、进行中课程数
- **继续学习**：最近在学习中的课程列表，带进度条
- **近期考试**：最近考试记录及得分
- **热门课程**：学习人数最多的课程排行

### 2.2 学习中心

#### 课程列表
- 按「知识领域」筛选（9 个领域）
- 按「难度等级」筛选（入门 L1 / 进阶 L2 / 高级 L3）
- 每门课显示：等级标签、所属领域、阅读时长、学习状态（未开始 / 进行中 / 已完成）

#### 课程详情
- Markdown 渲染正文，支持代码块、表格
- 课程信息：领域、等级、阅读时长
- **标记为已完成** 按钮，自动追踪学习进度
- **课后练习**：每个课程附带 5 道练习题（不计入考试成绩），提交后显示正确答案对比

### 2.3 题库管理（管理员）

#### 三种题型
| 题型 | 选项 | 答案格式 | 分值范围 |
|------|------|------|------|
| 单选题 | 4 个选项（A/B/C/D） | 单个字母 | 2-5 分 |
| 多选题 | 4-6 个选项 | 多个字母（如 A,B,C） | 3-6 分 |
| 判断题 | 无选项 | True / False | 1-2 分 |

#### 题目属性
- 所属知识领域（9 选 1）
- 难度等级（L1/L2/L3）
- 题型、题干、选项、正确答案、分值

#### 操作
- 添加题目（表单式创建，题型切换自动改变选项和答案输入方式）
- 编辑题目
- 删除题目（二次确认）
- 筛选：领域 / 等级 / 题型 / 关键词搜索

### 2.4 考试中心

#### 创建考试（管理员）

**三种组卷方式：**

| 方式 | 操作 |
|------|------|
| **📋 手动选题** | 按领域/等级筛选 → 逐题勾选 → 也支持「全选」「清空」「随机抽取 N 题」快捷按钮 |
| **📊 按分布选题** | 展示 9 领域 × 3 等级的选题分布表格，每格输入抽题数 → 点击「应用选题」随机抽取 |
| **🤖 智能组卷** | 设定目标总分（默认 100）→ 系统按题库比例自动抽取，**精确匹配目标分数** |

考试属性：
- 考试名称
- 时长（分钟），带单位标签
- 及格分数，带单位标签
- 考试描述（可选）

#### 参加考试（用户）

- 考试列表：显示名称、题数、时长、总分、及格线、状态
- 已参加的考试显示「查看成绩」按钮，未参加显示「开始考试」
- **答题界面**：
  - 倒计时（红色闪烁提醒最后 1 分钟），超时自动交卷
  - 左侧答题卡：已答/未答颜色区分，点击跳转
  - 题型自适应：单选 radio、多选 checkbox、判断 正确/错误 切换
  - 上一题 / 下一题 导航
  - 交卷前提示未答题数量

#### 自动评分

交卷后即时出分，无需人工审核。

| 题型 | 评分规则 |
|------|------|
| 单选题 | 答案完全匹配即得满分 |
| 判断题 | 答案完全匹配即得满分，支持 `true`/`false`/`1`/`0`/`"true"`/`"false"` 等格式 |
| 多选题 | 全部选对得满分，部分选对得一半分，有错选不得分 |

#### 考试结果

- 得分卡片：总分、及格线、通过/未通过
- 逐题详情：题目、你的答案、正确答案、得分，正确/错误颜色区分

### 2.5 成绩分析

#### 个人维度
- 统计概览：考试次数、平均分、最高分
- **考试趋势**：每次考试得分柱状图
- **知识点掌握度**：9 个领域分别的正确率进度条
- **错题本**：所有错题回顾，可展开查看正确答案

#### 部门维度（管理员）
- 学员总数、考试总次数、部门平均分、通过率
- 各等级（L1/L2/L3）正确率对比
- 各知识领域薄弱环节排行（正确率从低到高）

### 2.6 用户管理（管理员）

- 用户列表：用户名、姓名、部门、角色、操作
- 添加用户：用户名、密码、姓名、部门、角色（普通用户/管理员）
- 编辑用户：姓名、部门、角色
- 删除用户：保护最后一个管理员不可删除

## 五、系统设计

### 5.1 数据模型

```
users              — 用户表（username, password, real_name, department, role, is_active）
categories          — 知识领域（name, description, sort_order）
courses             — 课程表（title, category_id, level, content, read_time_minutes）
course_progress     — 学习进度（user_id, course_id, status）
questions           — 题库（category_id, level, type, title, options[JSON], correct_answer[JSON], score）
exams               — 考试定义（title, description, time_limit_minutes, pass_score, total_score, status）
exam_questions      — 考试-题目关联（exam_id, question_id, sort_order）
exam_submissions    — 提交记录（exam_id, user_id, score, status, start_time, submit_time）
exam_answers        — 答题详情（submission_id, question_id, user_answer[JSON], is_correct, score_earned）
```

### 5.2 API 结构

| 路由 | 说明 |
|------|------|
| `/api/auth/*` | 登录、获取用户信息、修改密码 |
| `/api/users/*` | 用户 CRUD（管理员） |
| `/api/courses/*` | 课程 CRUD + 分类列表 + 标记完成 |
| `/api/questions/*` | 题库 CRUD（管理员） |
| `/api/exams/*` | 考试创建/列表/答题/提交/查看结果 |
| `/api/stats/*` | 首页统计/个人分析/部门分析 |

### 5.3 前端路由

| 路由 | 页面 |
|------|------|
| `/login` | 登录页 |
| `/` | 首页仪表盘 |
| `/courses` | 课程列表 |
| `/courses/:id` | 课程详情 |
| `/exams` | 考试列表 |
| `/exams/:id` | 答题页 |
| `/exams/:id/result` | 考试成绩 |
| `/stats` | 个人成绩分析 |
| `/stats/department` | 部门成绩分析（管理员） |
| `/admin/questions` | 题库管理（管理员） |
| `/admin/courses` | 课程管理（管理员） |
| `/admin/users` | 用户管理（管理员） |
| `/admin/exams` | 考试管理（管理员） |

## 七、非功能需求

1. **易迁移**：SQLite 单文件数据库，直接拷贝 `backend/ai_study.db` 即可迁移
2. **零外部依赖**：不依赖任何外部 AI API，评分逻辑完全本地化
3. **响应式安全**：JWT 7 天过期，密码 bcrypt 加密，管理员路由权限守卫
4. **开发体验**：FastAPI 自动生成 Swagger 文档（`/docs`），Vite HMR 热更新
